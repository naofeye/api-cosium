Overview

HTTP verbs

Cosium Center APIs try to adhere as closely as possible to standard HTTP and REST conventions in its use of HTTP verbs.

Verb Usage

GET

Used to retrieve a resource

POST

Used to create a new resource

PUT

Used to create or update a resource in an idempotent way

PATCH

Used to update an existing resource, including partial updates

DELETE

Used to delete an existing resource

HTTP status codes

Cosium Center APIs tries to adhere as closely as possible to standard HTTP and REST conventions in its use of HTTP status codes.

Status code Usage

200 OK

The request completed successfully.

201 Created

A new resource has been created successfully. The resource’s URI is available from the response’s Location header.

204 No Content

A resource update or deletion has been applied successfully.

400 Bad Request

The request was malformed. The response body will include an error providing further information.

401 Unauthorized

The request could not be authenticated. You must provide a valid access token.

403 Forbidden

The request has been authenticated but the operation is forbidden. This can be caused by insufficient privileges.

404 Not Found

The requested resource did not exist.

API Sunset

The Sunset header field contains a single timestamp that advertises the point in time when the resource is expected to become unresponsive.
— RFC 8594 The Sunset HTTP Header Field
The Sunset HTTP Response Header Field

Example:

Sunset: 01 Aug 2025 00:00:00 GMT

Authentication

Authentication must be done via the OpenID Connect (OIDC) protocol. In accordance to OIDC, the consumer (your http client) must authenticate itself to the OIDC issuer.

Application OIDC client ID

Each access token retrieval request should contain the intended recipient of the token. In OIDC, the intended recipient of a token is designated as the audience. The value of an audience is an OIDC client ID. To designate a Cosium Center APIs as the intended audience of the token, you must first retrieve its OIDC client id.

Example request

GET /01CONE06488/api/application-oidc-client-id HTTP/1.1
Host: c1.cosium.biz

Example response

HTTP/1.1 200 OK
Vary: Origin
Vary: Access-Control-Request-Method
Vary: Access-Control-Request-Headers
Vary: Cookie
Vary: Authorization
Vary: Prefer
Content-Type: application/json
Server-Timing: app;desc="App {'thread-name':'00000000000-uid_-1@main'}";dur=36
Cache-Control: no-cache, no-store, max-age=0, must-revalidate
Pragma: no-cache
Expires: 0
Strict-Transport-Security: max-age=31536000 ; includeSubDomains
Content-Length: 72

{
  "applicationOidcClientId" : "d18a70c1-17bb-4435-b4ca-1aca36148ed9"
}

The value of applicationOidcClientId must be passed as a requested scope in each access token retrieval request.

e.g. If the value of applicationOidcClientId is 474a698c-9511-11ee-b9d1-0242ac120002, you will need to add the form request parameter scope=audience:474a698c-9511-11ee-b9d1-0242ac120002 (URL encoded as scope=audience%3A474a698c-9511-11ee-b9d1-0242ac120002) to the access token retrieval request.

Any access token retrieved without intended audience will be rejected by the Cosium Center APIs.

Access token retrieval

Knowing the OIDC issuer configuration and the application OIDC client ID, the consumer can authenticate to the issuer by using one of the 2 grant flows:

  • Client Credentials

  • Authorization code

To use any of these flows, a consumer will need to authenticate itself as an OIDC client to the OIDC issuer.

Client Credentials flow

In this scenario, the consumer authenticates itself using its OIDC client credentials.

Most of the time, an OIDC client credentials consists of a Client ID and a Client Secret. In this case, the authentication request looks like this:

Accept: application/json
Host: example.org

Authorization code flow

Authorization code must be used when the consumer wants to impersonate a user account. When triggered, this flow will redirect the user account owner to the OIDC issuer to let him enter its credentials. Only if the owner authenticates successfully to the issuer and the owner explicitly allows the consumer to use a token impersonating the former, then the consumer will receive an access token that can be used to interact with Cosium Center APIs.

In this scenario, the consumer will have at most the same permissions as the impersonated user.

Implementing this flow is difficult. Therefore, we recommend consumers to use this flow via a certified OIDC library.

Hypermedia

Cosium Center APIs uses hypermedia. Responses are in HAL-Forms format. Links can be found beneath the _links key. Affordances can be found beneath the _templates key.

Consumers of the API should not handcraft URIs themselves, instead they should use the above-described links to navigate.

To obtain HAL-Forms representation, the consumer must provide header Accept: application/prs.hal-forms+json to its HTTP requests.

Resources

Get the link for the After Sales Service API

Now that you are authenticated, you have to get the link of the after sales service API. Make a GET Http request to the URI : http://c1.cosium.biz/01CONE06488/api and then you will get the URI by following the link : list-after-sales-service .

Get After Sales Service filtered list

Example request

GET /01CONE06488/api/after-sales-services?status=TO_REPAIR&creation_date=2026-04-09&site_name=TEST&type=INTERNAL&repairer_name=Mr.Test HTTP/1.1
Accept: application/prs.hal-forms+json
Host: c1.cosium.biz

status

Status of the after sales service. Possible values : FINISHED, IN_PROCESS, REPAIR_IN_PROCESS, TO_REPAIR

resolution_status

Resolution status of the after sales service. Possible values : RESOLVED, SOLD_OUT

creation_date

Creation date of the after sales service. Format : "yyyy-mm-dd"

processing_date

Processing date of the after sales service. Format : "yyyy-mm-dd"

site_name

Site name of the site associated to the after sales service

type

Type of the after sales service. Possible values : EXTERNAL, INTERNAL

repairer_name

Name of the repairer that handles the after sales service

All parameters are optional, mind the cases and the date format, otherwise you’ll receive an error. You can choose the size of your pagination result by using the parameter 'page_size'.
